## Starter Template

The **starter template** is a comprehensive starting point with examples of all MCP features.

### Use Case

Best for:
- Learning MCP concepts
- Exploring all features
- Prototyping quickly
- Building full-featured servers

### What's Included

<Tabs>
  <Tab title="Tools">
    Example tool implementations:

    ```typescript src/index.ts
    // Weather fetching tool
    server.tool(
      {
        name: "fetch-weather",
        description: "Fetch weather for a city",
        schema: z.object({
          city: z.string().describe("City name"),
        }),
      },
      async ({ city }) => {
        return text(`Weather in ${city}: Sunny, 72°F`);
      }
    );
    ```
  </Tab>

  <Tab title="Resources">
    Example resource definitions:

    ```typescript src/index.ts
    // Configuration resource
    server.resource(
      {
        name: "config",
        uri: "config://settings",
        description: "Server configuration",
      },
      async () => object({
        theme: "dark",
        language: "en",
      })
    );
    ```
  </Tab>

  <Tab title="Prompts">
    Example prompt templates:

    ```typescript src/index.ts
    // Code review prompt
    server.prompt(
      {
        name: "review-code",
        description: "Review code for best practices",
        schema: z.object({
          language: z.string(),
          code: z.string(),
        }),
      },
      async ({ language, code }) => {
        return text(`
Reviewing ${language} code:

${code}

Check for:
- Best practices
- Security issues
- Performance problems
        `);
      }
    );
    ```
  </Tab>

  <Tab title="Widgets">
    Example React widgets:

    ```tsx resources/display-weather.tsx
    import React from "react";
    import { useMcp } from "mcp-use/react";
    import { z } from "zod";

    const propSchema = z.object({
      city: z.string(),
      temperature: z.number(),
    });

    export const widgetMetadata = {
      description: "Display weather information",
      props: propSchema,
    };

    export default function WeatherDisplay() {
      const { props } = useMcp<z.infer<typeof propSchema>>();

      return (
        <div className="p-4">
          <h1 className="text-2xl font-bold">{props.city}</h1>
          <p className="text-lg">{props.temperature}°F</p>
        </div>
      );
    }
    ```
  </Tab>
</Tabs>

## MCP Apps Template

The **mcp-apps template** is optimized for OpenAI Apps SDK integration and ChatGPT compatibility.

### Use Case

Best for:
- OpenAI ChatGPT integration
- Apps SDK widgets
- Ecommerce applications
- ChatGPT-compatible UIs

### What's Included

<Tabs>
  <Tab title="OpenAI Widgets">
    Pre-built OpenAI-compatible widgets:

    **1. Display Weather**
    ```tsx resources/display-weather.tsx
    // Weather card with theme support
    import { useWidget } from 'mcp-use/react';

    export default function DisplayWeather() {
      const { props, theme } = useWidget();
      // theme is 'dark' or 'light' from ChatGPT
      return (
        <div className={theme === 'dark' ? 'bg-gray-800' : 'bg-white'}>
          {/* ... */}
        </div>
      );
    }
    ```

    **2. Ecommerce Carousel**
    ```tsx resources/ecommerce-carousel.tsx
    // Product carousel with OpenAI SDK UI components
    import { Card, Button } from '@openai/apps-sdk-ui';

    export default function EcommerceCarousel() {
      const { props, sendFollowUpMessage } = useWidget();

      return (
        <div className="carousel">
          {props.products.map(product => (
            <Card key={product.id}>
              <img src={product.image} />
              <h3>{product.name}</h3>
              <Button onClick={() => sendFollowUpMessage(
                `Tell me more about ${product.name}`
              )}>
                Learn More
              </Button>
            </Card>
          ))}
        </div>
      );
    }
    ```

    **3. Product Search**
    ```tsx resources/product-search-result.tsx
    // Search results with filters
    ```

    **4. Store Locations Map**
    ```tsx resources/stores-locations-map.tsx
    // Interactive map showing store locations
    ```

    **5. Order Confirmation**
    ```tsx resources/order-confirmation.tsx
    // Order summary and confirmation
    ```
  </Tab>

  <Tab title="Brand Info Tool">
    Example API tool:

    ```typescript src/index.ts

    server.get("/api/fruits", (c) => {
      return c.json([
        { fruit: "mango", color: "bg-[#FBF1E1]" },
        { fruit: "apple", color: "bg-[#F9E7E4]" },
        // ...
      ]);
    });

    // Brand information tool

      {
        name: "get-brand-info",
        description: "Get brand information",

          name: "mcp-use",
          tagline: "Build MCP servers with UI widgets",

            "Automatic widget registration",
            "React component support",
    ```

  </Tab>

  <Tab title="Theme Integration">
    Theme detection from ChatGPT:

    ```tsx

      const { theme } = useWidget();
      // theme is 'dark' or 'light'

      return (
        <div className={`
          ${theme === 'dark' 
            ? 'bg-gray-900 text-white' 
            : 'bg-white text-gray-900'
          }
        `}>
          Content adapts to ChatGPT theme
        </div>
      );
    }
    ```
  </Tab>

  <Tab title="Widget Capabilities">
    Full Apps SDK features:

    ```tsx
    import { useWidget } from 'mcp-use/react';

    export default function MyWidget() {
      const { 
        props,              // Validated widget props
        theme,              // 'dark' | 'light'
        callTool,           // Call MCP tools
        sendFollowUpMessage, // Send message to chat
      } = useWidget();

      const handleClick = async () => {
        // Call MCP tool
        const result = await callTool('my-tool', {
          param: 'value',
        });

        // Send follow-up message
        sendFollowUpMessage(
          `Tool returned: ${result.content}`
        );
      };

      return <button onClick={handleClick}>Execute</button>;
    }
    ```
  </Tab>

## Blank Template

The **blank template** provides a minimal starting point for custom projects.

### Use Case

Best for:
- Custom implementations
- Minimal dependencies
- Learning from scratch
- Specific use cases

### What's Included

<Tabs>
  <Tab title="Server Only">
    Minimal server setup:

    ```typescript src/index.ts
    import { MCPServer } from "mcp-use/server";

    const server = new MCPServer({
      name: "my-server",
      version: "1.0.0",
    });

    // Add your tools, resources, and prompts here

    server.listen(3000);
    ```

    No example implementations - clean slate.
  </Tab>

  <Tab title="Configuration">
    Basic configuration files:

    ```json package.json
    {
      "name": "my-mcp-server",
      "version": "1.0.0",
      "type": "module",
      "scripts": {
        "dev": "mcp-use dev",
        "build": "mcp-use build",
        "start": "mcp-use start"
      },
      "dependencies": {
        "mcp-use": "^2.0.0"
      },
      "devDependencies": {
        "@mcp-use/cli": "^2.0.0",
        "typescript": "^5.0.0"
      }
    }
    ```

    ```json tsconfig.json
    {
      "compilerOptions": {
        "target": "ES2020",
        "module": "ESNext",
        "strict": true,
        "outDir": "./dist"
      }
    }
    ```
  </Tab>

  <Tab title="Structure">
    Minimal project structure:

    ```
    my-project/
    ├── package.json
    ├── tsconfig.json
    ├── .gitignore
    ├── README.md
    ├── src/
    │   └── index.ts        # Empty server
    ├── public/            # Empty
    └── resources/         # Empty
    ```

    Add your own implementations.
  </Tab>
</Tabs>

## Custom GitHub Templates

Use any GitHub repository as a template.

### Syntax

<CodeGroup>

```bash Short Format
npx create-mcp-use-app my-project --template owner/repo

# Example:
npx create-mcp-use-app my-project --template acme/mcp-template
```

```bash Full URL
npx create-mcp-use-app my-project --template https://github.com/owner/repo

# Example:
npx create-mcp-use-app my-project --template https://github.com/acme/mcp-template
```

```bash With Branch
npx create-mcp-use-app my-project --template owner/repo#branch-name

# Example:
npx create-mcp-use-app my-project --template acme/mcp-template#develop
```

</CodeGroup>

### Requirements

<Warning>
  Git must be installed and available in your PATH to use GitHub templates.
</Warning>

<Steps>
  <Step title="Public Repository">
    Repository must be publicly accessible (or you must have access).
  </Step>

  <Step title="Valid Structure">
    Repository should contain:
    - `package.json`
    - `src/` directory (or TypeScript files)
    - Optional: `tsconfig.json`, `README.md`
  </Step>

  <Step title="Template Variables">
    Files can include placeholders:
    - `{{PROJECT_NAME}}` - Replaced with your project name
    - `{{mcp-use_version}}` - Replaced with latest mcp-use version
  </Step>
</Steps>

### Example Custom Template

Create your own template repository:

```json package.json
{
  "name": "{{PROJECT_NAME}}",
  "version": "1.0.0",
  "dependencies": {
    "mcp-use": "{{mcp-use_version}}"
  }
}
```

```typescript src/index.ts
// Your custom template code
import { MCPServer } from "mcp-use/server";

const server = new MCPServer({
  name: "{{PROJECT_NAME}}",
  version: "1.0.0",
});

// Your custom tools and resources

server.listen(3000);
```

Users can then use:
```bash
npx create-mcp-use-app my-project --template your-org/your-template
```


  </Step>
