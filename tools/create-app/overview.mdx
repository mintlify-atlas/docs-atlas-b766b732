# Create mcp-use App

**create-mcp-use-app** is the fastest way to scaffold a new MCP (Model Context Protocol) application. With just one command, you get a fully configured TypeScript project with hot reload, automatic inspector, and UI widget support.

## Quick Start

Create a new MCP application in seconds:

```bash
npx create-mcp-use-app my-mcp-server
cd my-mcp-server
npm run dev
```

That's it! Your MCP server is running at `http://localhost:3000` with the inspector automatically opened.

## Installation Methods

<Tabs>
  <Tab title="Interactive">
    Run without arguments for interactive prompts:

    ```bash
    npx create-mcp-use-app
    ```

    You'll be prompted for:
    - Project name

    - Install dependencies
    - Skills installation (Claude Code/Cursor)

    Specify project name directly:

## Command Options

### Template Selection

```bash
# Use starter template (default)
npx create-mcp-use-app my-project --template starter

# Use mcp-apps template
npx create-mcp-use-app my-project --template mcp-apps

# Use blank template
npx create-mcp-use-app my-project --template blank

# Use GitHub repository as template
npx create-mcp-use-app my-project --template owner/repo
npx create-mcp-use-app my-project --template https://github.com/owner/repo
```

### Package Manager

```bash
# Use npm (default)
npx create-mcp-use-app my-project --npm

# Use yarn
npx create-mcp-use-app my-project --yarn

# Use pnpm
npx create-mcp-use-app my-project --pnpm
```

### Dependency Installation

```bash
# Install dependencies automatically
npx create-mcp-use-app my-project --install

# Skip installation
npx create-mcp-use-app my-project --no-install
```

### Skills Installation

```bash
# Install skills (prompts for Claude Code/Cursor/Both)
npx create-mcp-use-app my-project --skills

# Skip skills installation
npx create-mcp-use-app my-project --no-skills
```

### List Templates

```bash
# Show all available templates
npx create-mcp-use-app --list-templates
```

    MCP server with example implementations:

    ```typescript
    import { MCPServer, text, object } from "mcp-use/server";
    import { z } from "zod";

    const server = new MCPServer({
      name: "my-mcp-server",
      version: "1.0.0",
      description: "My first MCP server",
    });

    // Example tool
    server.tool(
      {
        name: "fetch-weather",
        description: "Fetch weather for a city",
        schema: z.object({
          city: z.string(),
        }),
      },
      async ({ city }) => {
        return text(`Weather in ${city}: Sunny`);
      }
    );

    // Example resource
    server.resource(
      {
        name: "config",
        uri: "config://settings",
      },
      async () => object({ theme: "dark" })
    );

    server.listen(3000);
    ```

  </Step>
</Steps>

## Skills Integration

Optionally install AI coding assistant skills:

<Info>
  Skills provide context and instructions for AI coding assistants like Claude Code and Cursor.
</Info>

### What Are Skills?

Skills are markdown files (`.ai/` or `.cursorrules`) that give AI assistants:
- Project-specific context
- Code style guidelines
- Common patterns
- Framework documentation

### Installation Options

During project creation, choose:

<Tabs>
  <Tab title="Claude Code">
    Installs `.ai/mcp-use.md` with:
    - mcp-use framework documentation
    - TypeScript patterns
    - Best practices
  </Tab>

  <Tab title="Cursor">
    Installs `.cursorrules/mcp-use.md` for Cursor IDE.
  </Tab>

  <Tab title="Both">
    Installs skills for both assistants.
  </Tab>

  <Tab title="None">
    Skips skills installation.
  </Tab>
</Tabs>

## Package Manager Detection

The CLI automatically detects your package manager:

```bash
# Detects from npm_config_user_agent
npm create mcp-use-app my-project  # Uses npm
yarn create mcp-use-app my-project # Uses yarn  
pnpm create mcp-use-app my-project # Uses pnpm
```

Or explicitly specify:

```bash
npx create-mcp-use-app my-project --pnpm
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Command Not Found" icon="circle-xmark">
    **Error:** `create-mcp-use-app: command not found`

    **Solutions:**
    
    1. Ensure Node.js 20.19+ is installed:
       ```bash
       node --version
       ```

    2. Use npx (no installation needed):
       ```bash
       npx create-mcp-use-app@latest my-project
       ```

    3. Try with explicit registry:
       ```bash
       npm config set registry https://registry.npmjs.org/
       npx create-mcp-use-app my-project
       ```
  </Accordion>

  <Accordion title="Template Not Found" icon="folder">
    **Error:** `Template 'xyz' not found`

    **Solutions:**
    
    1. List available templates:
       ```bash
       npx create-mcp-use-app --list-templates
       ```

    2. Use valid template name:
       ```bash
       npx create-mcp-use-app my-project --template starter
       ```

    3. For GitHub templates, check repository exists:
       ```bash
       curl -I https://github.com/owner/repo
       ```
  </Accordion>

  <Accordion title="Permission Denied" icon="lock">
    **Error:** `EACCES: permission denied`

    **Solutions:**
    
    1. Check directory permissions:
       ```bash
       ls -la
       ```

    2. Create in user directory:
       ```bash
       cd ~
       npx create-mcp-use-app my-project
       ```

    3. On macOS/Linux, avoid sudo with npm:
       ```bash
       # Don't use:
       sudo npx create-mcp-use-app
       ```
  </Accordion>

  <Accordion title="Installation Fails" icon="download">
    **Error:** Installation hangs or fails

    **Solutions:**
    
    1. Clear npm cache:
       ```bash
       npm cache clean --force
       ```

    2. Try different package manager:
       ```bash
       npx create-mcp-use-app my-project --yarn
       ```

    3. Install manually:
       ```bash
       npx create-mcp-use-app my-project --no-install
       cd my-project
       npm install
       ```
  </Accordion>
</AccordionGroup>
